# Startup Frameworks Reference

Use these frameworks when structuring advice for founders. Reference them by name and adapt to the founder's stage.

## Lean Canvas
A one-page business model summary. Sections: Problem, Solution, Key Metrics, Unique Value Proposition, Unfair Advantage, Channels, Customer Segments, Cost Structure, Revenue Streams. Best for early-stage, pre-product-market-fit startups.

## Business Model Canvas
Broader than Lean Canvas, covers 9 building blocks: Key Partners, Key Activities, Key Resources, Value Propositions, Customer Relationships, Channels, Customer Segments, Cost Structure, Revenue Streams. Better suited for more established businesses.

## AARRR (Pirate Metrics)
Framework for growth and funnel analysis: Acquisition, Activation, Retention, Referral, Revenue. Useful when a founder asks about growth, funnel drop-off, or metrics to track.

## SWOT Analysis
Strengths, Weaknesses (internal), Opportunities, Threats (external). Good for quick strategic assessments or competitor comparisons.

## ICE / RICE Prioritization
- ICE: Impact, Confidence, Ease — score each 1-10, multiply for a priority score.
- RICE: Reach, Impact, Confidence, Effort — (Reach x Impact x Confidence) / Effort.
Use when a founder needs to prioritize features, experiments, or initiatives.

## Fundraising Stages (rough guide)
- Pre-seed: idea/prototype, founder's own network, angels
- Seed: early traction, MVP live, seed VCs and angels
- Series A: proven product-market fit, repeatable growth, institutional VCs
- Series B+: scaling, expansion, later-stage VCs

## MVP Validation Checklist
1. Clearly defined problem and target customer
2. Smallest feature set that delivers core value
3. Way to measure whether the value was delivered (metric or feedback loop)
4. Fast iteration cycle based on real user feedback
